



# Linux — Processes & Jobs

---

## What a Process Is

Every running program is a process. Each has:

|Property|Meaning|
|---|---|
|PID|Process ID — unique number|
|PPID|Parent process ID|
|UID|User who owns it|
|State|Running, sleeping, stopped, zombie|
|Nice|Priority (-20 highest → 19 lowest)|

---

## Viewing Processes

```bash
ps aux              # snapshot of all running processes
ps aux | grep name  # find specific process
pgrep python        # get PID by name
pidof nginx         # get PID of program

top                 # live process monitor
htop                # better top — interactive (install if needed)
```

### ps aux columns

```
USER   PID  %CPU  %MEM  VSZ   RSS  TTY  STAT  START  TIME  COMMAND
root     1   0.0   0.1  ...   ...  ?    Ss    ...    ...   /sbin/init
```

|Column|Meaning|
|---|---|
|%CPU|CPU usage|
|%MEM|Memory usage|
|VSZ|Virtual memory size|
|RSS|Actual RAM used|
|STAT|Process state (S=sleep, R=run, Z=zombie)|

## Killing Processes

```bash
kill PID              # send SIGTERM (graceful stop)
kill -9 PID           # send SIGKILL (force stop — no cleanup)
killall name          # kill all processes by name
pkill -f pattern      # kill by pattern match on command
```

|Signal|Number|Meaning|
|---|---|---|
|SIGTERM|15|Graceful shutdown (default)|
|SIGKILL|9|Force kill — cannot be caught|
|SIGHUP|1|Reload config|
|SIGINT|2|Interrupt (Ctrl+C)|
|SIGSTOP|19|Pause process|

**Always try SIGTERM before SIGKILL.**

## Background & Foreground Jobs

```bash
command &           # run in background
Ctrl+Z              # suspend foreground job
jobs                # list background/suspended jobs
fg                  # bring last job to foreground
fg %2               # bring job 2 to foreground
bg                  # resume suspended job in background
bg %2               # resume job 2 in background
```

```bash
# example
python train.py &   # start in background
jobs                # [1]+ Running  python train.py
fg %1               # bring back to foreground
```

---

## nohup — Keep Running After Logout

```bash
nohup command &                     # run immune to hangup
nohup python train.py > out.log &   # redirect output to file
```

Output goes to `nohup.out` by default if not redirected.  
Process keeps running after you close the terminal.

## tmux — Persistent Sessions

Better than nohup for interactive work.

```bash
tmux                        # start new session
tmux new -s name            # named session
tmux ls                     # list sessions
tmux attach -t name         # reattach to session
tmux kill-session -t name   # kill session
```

|Shortcut (inside tmux)|Action|
|---|---|
|`Ctrl+B d`|Detach (leave running)|
|`Ctrl+B c`|New window|
|`Ctrl+B n / p`|Next / previous window|
|`Ctrl+B %`|Split vertical|
|`Ctrl+B "`|Split horizontal|
|`Ctrl+B arrow`|Move between panes|

## Process Priority

```bash
nice -n 10 command          # start with lower priority (10)
renice -n 5 -p PID          # change priority of running process
```

Range: -20 (highest priority) to 19 (lowest).  
Only root can set negative nice values.

## systemd — Service Management

```bash
sudo systemctl start service      # start
sudo systemctl stop service       # stop
sudo systemctl restart service    # restart
sudo systemctl status service     # check status
sudo systemctl enable service     # start on boot
sudo systemctl disable service    # don't start on boot
sudo systemctl list-units         # list all services

journalctl -u service             # logs for a service
journalctl -f                     # follow all logs live
journalctl -n 100                 # last 100 log lines
```

## cron — Scheduled Jobs

```bash
crontab -e      # edit cron jobs for current user
crontab -l      # list cron jobs
crontab -r      # remove all cron jobs
```

### Cron syntax

```
* * * * * command
│ │ │ │ └── day of week (0=Sun, 6=Sat)
│ │ │ └──── month (1-12)
│ │ └────── day of month (1-31)
│ └──────── hour (0-23)
└────────── minute (0-59)
```

```bash
# examples
0 2 * * *   /backup.sh          # every day at 2am
*/5 * * * * python monitor.py   # every 5 minutes
0 9 * * 1   report.sh           # every Monday at 9am
```

## Monitoring Resources

```bash
free -h             # RAM usage
vmstat 1            # CPU, memory, IO every 1 sec
iostat              # disk IO stats
lsof -p PID         # files opened by process
lsof -i :8080       # process using port 8080
watch -n 2 command  # run command every 2 seconds
```

---

## Common Patterns

```bash
# find and kill process on a port
lsof -i :8080
kill -9 $(lsof -t -i:8080)

# run long training job safely
nohup python train.py > train.log 2>&1 &
tail -f train.log         # watch progress

# check if process is still alive
kill -0 PID && echo "running" || echo "dead"

# run at low priority to not slow the system
nice -n 19 python heavy_script.py
```
