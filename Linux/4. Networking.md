



# Linux — Networking

---

## Basic Network Info

```bash
ip a                    # show all interfaces and IPs
ip r                    # routing table
hostname -I             # just the IP address
ping google.com         # test connectivity
ping -c 4 google.com    # ping 4 times then stop
traceroute google.com   # trace route to host
```

## SSH

```bash
ssh user@host                       # connect
ssh -p 2222 user@host               # custom port
ssh -i ~/.ssh/key.pem user@host     # specific key
ssh -L 8080:localhost:80 user@host  # local port forwarding
ssh -R 9090:localhost:9090 user@host # remote port forwarding
```

### SSH Config (~/.ssh/config)

Avoid typing long commands — define aliases:

```
Host myserver
    HostName 192.168.1.10
    User ubuntu
    Port 22
    IdentityFile ~/.ssh/id_rsa
```

```bash
ssh myserver    # uses config above
```

### SSH Key Setup

```bash
ssh-keygen -t ed25519 -C "email"    # generate key pair
ssh-copy-id user@host               # copy public key to server
chmod 600 ~/.ssh/id_rsa             # secure private key
cat ~/.ssh/id_rsa.pub               # view public key
```

## SCP & rsync — File Transfer

```bash
# scp — simple copy over SSH
scp file.txt user@host:/remote/path
scp user@host:/remote/file .        # download
scp -r dir/ user@host:/remote/      # directory

# rsync — smarter, faster, resumable
rsync -avz dir/ user@host:/remote/  # sync local to remote
rsync -avz user@host:/remote/ dir/  # sync remote to local
rsync -avz --delete dir/ user@host:/remote/  # mirror (delete extra)
rsync -avz --progress large_file user@host:/path  # show progress
```

|Flag|Meaning|
|---|---|
|`-a`|Archive — preserve permissions, timestamps|
|`-v`|Verbose|
|`-z`|Compress during transfer|
|`--delete`|Remove files on dest not in source|
|`--dry-run`|Preview without executing|

## curl — HTTP Requests

```bash
curl https://example.com                        # GET request
curl -o file.html https://example.com           # save to file
curl -I https://example.com                     # headers only
curl -X POST -d '{"key":"val"}' url             # POST JSON
curl -H "Authorization: Bearer token" url       # with header
curl -u user:pass https://example.com           # basic auth
curl -L url                                     # follow redirects
curl -s url | jq .                              # silent + parse JSON
```

## wget — Download Files

```bash
wget https://example.com/file.zip               # download file
wget -O newname.zip url                         # custom filename
wget -r -np url                                 # recursive download
wget -c url                                     # resume interrupted download
wget -q url                                     # quiet mode
```

## Ports & Connections

```bash
ss -tulnp               # listening ports + process (modern)
netstat -tulnp          # same (older systems)
lsof -i :8080           # what's using port 8080
lsof -i tcp             # all TCP connections

# check if port is open on remote host
nc -zv host 8080
telnet host 8080
```

|Column (ss)|Meaning|
|---|---|
|`t/u`|TCP / UDP|
|`l`|Listening only|
|`n`|Show numbers not names|
|`p`|Show process|

## Firewall — ufw

```bash
sudo ufw status                 # check status
sudo ufw enable                 # turn on
sudo ufw disable                # turn off

sudo ufw allow 22               # allow SSH
sudo ufw allow 80               # allow HTTP
sudo ufw allow 443              # allow HTTPS
sudo ufw deny 8080              # block port
sudo ufw allow from 192.168.1.0/24  # allow subnet
sudo ufw delete allow 80        # remove rule
```

## DNS

```bash
nslookup google.com             # query DNS
dig google.com                  # detailed DNS lookup
dig google.com +short           # just the IP
host google.com                 # simple lookup
cat /etc/resolv.conf            # DNS server config
cat /etc/hosts                  # local hostname overrides
```


## Common Patterns

```bash
# tunnel a remote Jupyter notebook to local browser
ssh -L 8888:localhost:8888 user@host
# then open localhost:8888 in browser

# copy SSH key to new server
ssh-copy-id -i ~/.ssh/id_rsa.pub user@host

# check what's blocking a port
sudo lsof -i :80

# download a large file and resume if interrupted
wget -c https://example.com/large.tar.gz

# send a file securely and fast
rsync -avz --progress model.pt user@host:~/models/

# test if a service is responding
curl -f http://localhost:8080/health || echo "service down"
```
