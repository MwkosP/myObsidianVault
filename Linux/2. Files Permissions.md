



# Linux — Files & Permissions

---

## Permission Structure

Every file has permissions for three groups:

```
-rwxr-xr--  1  user  group  4096  Mar 1  file.sh
│└─┬──┘└─┬──┘└─┬──┘
│  │     │     └── others (everyone else)
│  │     └──────── group
│  └────────────── owner (user)
└───────────────── file type (- file, d dir, l symlink)
```

Each group has three bits:

|Bit|Symbol|Value|Meaning|
|---|---|---|---|
|Read|`r`|4|View contents|
|Write|`w`|2|Modify contents|
|Execute|`x`|1|Run as program|
|None|`-`|0|No permission|

---

## Reading Permissions

```
rwxr-xr--
│││├──┤└─┬─┘
│││  │   └── others: r-- = 4 (read only)
│││  └────── group:  r-x = 5 (read + execute)
│└┴────────── owner:  rwx = 7 (full)
```

Numeric = sum of bits per group:

|Permission|Numeric|
|---|---|
|`rwx`|7|
|`rw-`|6|
|`r-x`|5|
|`r--`|4|
|`---`|0|

So `rwxr-xr--` = **754**

---

## chmod — Change Permissions

```bash
chmod 755 file        # rwxr-xr-x  — owner full, others read+exec
chmod 644 file        # rw-r--r--  — owner rw, others read
chmod 600 file        # rw-------  — owner only (private keys)
chmod +x file         # add execute for everyone
chmod -w file         # remove write for everyone
chmod u+x file        # add execute for owner only
chmod g-w file        # remove write from group
chmod -R 755 dir/     # apply recursively
```

|Common|Numeric|Use|
|---|---|---|
|`rwxr-xr-x`|755|Scripts, directories|
|`rw-r--r--`|644|Regular files|
|`rw-------`|600|SSH keys, secrets|
|`rwx------`|700|Private directories|

---

## chown — Change Ownership

```bash
chown user file           # change owner
chown user:group file     # change owner and group
chown -R user:group dir/  # recursive
chown :group file         # change group only
```

## sudo — Run as Root

```bash
sudo command              # run single command as root
sudo -i                   # open root shell
sudo -u otheruser cmd     # run as another user
sudo !!                   # re-run last command as root
```

```bash
# edit sudoers file safely
sudo visudo
```

**Never run as root by default** — use sudo only when needed.

## Special Permissions

|Permission|Symbol|Meaning|
|---|---|---|
|Setuid|`s` on owner exec|Run as file owner, not caller|
|Setgid|`s` on group exec|Run as group of file|
|Sticky bit|`t` on others exec|Only owner can delete (e.g. `/tmp`)|

```bash
chmod u+s file    # setuid
chmod g+s dir     # setgid
chmod +t dir      # sticky bit
```

---

## umask — Default Permissions

Controls default permissions for newly created files.

```bash
umask           # show current umask (e.g. 022)
umask 027       # set new umask
```

New file permissions = `666 - umask`  
New dir permissions = `777 - umask`

|umask|Files|Dirs|
|---|---|---|
|022|644|755|
|027|640|750|
|077|600|700|

## File Attributes

```bash
ls -la              # show all files + permissions
stat file           # detailed file metadata
file filename       # detect file type
```

```
stat output:
  File: file.sh
  Size: 4096
  Access: 2026-03-01
  Modify: 2026-02-28
  Permissions: -rwxr-xr-x
```

## Users & Groups

```bash
whoami              # current user
id                  # uid, gid, groups
groups              # list groups of current user
cat /etc/passwd     # all users
cat /etc/group      # all groups

# add/remove users
sudo useradd -m username
sudo userdel -r username
sudo passwd username

# add user to group
sudo usermod -aG groupname username
```

---

## Common Patterns

```bash
# make script executable
chmod +x script.sh && ./script.sh

# secure a private key
chmod 600 ~/.ssh/id_rsa

# give group access to a directory
chown -R :mygroup /var/www
chmod -R 775 /var/www

# find files with dangerous permissions
find / -perm -4000 2>/dev/null    # setuid files
find / -perm -o+w 2>/dev/null     # world-writable files
```
