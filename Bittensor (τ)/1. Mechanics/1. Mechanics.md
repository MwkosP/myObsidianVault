


# Bittensor — Mechanics

---

## 1. Core Components

|Symbol|Name|What it is|
|---|---|---|
|`f(x)`|Peer function|A neural network model owned by one participant|
|`F = {f₀...fₙ}`|Peer set|All n models in the network|
|`D = [X, Y]`|Dataset|Input-output pairs a peer trains on|
|`L = E_D[Q(y, f(x))]`|Loss|How wrong a model's predictions are|
|`S = [sᵢ]`|Stake vector|Each peer's stake (reputation/weight) on-chain|
|`W = [wᵢⱼ]`|Weight matrix|Peer i's score of peer j's usefulness|
|`I`|Incentive vector|New stake distributed each round|
|`τ`|Emission rate|How many new tokens minted per round|

---

## 2. Network Objective

The network doesn't optimize one model — it optimizes a **stake-weighted sum of losses**:

$$\sum_{i}^{n} L_i \cdot s_i$$

- Peers with more stake have more influence on what "good" means
- Each peer has its own loss `Lᵢ` on its own dataset `Dᵢ`
- The network's collective goal is to minimize this weighted sum

---

## 3. Stake Update Rule

Each round, stake is updated by distributing new tokens proportional to incentive:

$$S_{t+1} = S_t + \tau I$$

- `τ` = emission rate (fixed token release per block)
- `I` = incentive vector — who gets what share
- Over time, stake flows toward peers that consistently earn high `I`

**The entire mechanism design problem is: how do you compute `I` honestly?**

---

## 4. Peer Ranking

### How peers evaluate each other

Instead of a central authority, peers evaluate each other:

- Each peer feeds all other peers' outputs into itself: `f(F(x))`
- Peer `i` learns weights `wᵢⱼ` — how much it values peer `j`
- These weights are submitted as transactions on-chain

### The ranking formula

$$R = W^T \cdot S$$

- `W` = weight matrix (n×n), peer votes
- `S` = current stake vector
- `R` = rank/incentive vector — the output score for each peer

**Intuition:** a vote from a high-stake peer counts more. If a trusted peer says you're useful, you earn more rank.

---

## 5. Ideal Score — Fisher Information Pruning

The theoretically correct score for peer `i` is its **pruning score**: how much does the network _lose_ if peer `i` is removed?

$$r_i \approx \frac{1}{n} \sum_{j}^{n} \sum_{x \in D_j} \Delta F^T(x)_i \cdot H(Q_j(x)) \cdot \Delta F(x)_i$$

| Term          | Meaning                                                          |
| ------------- | ---------------------------------------------------------------- |
| `ΔF(x)ᵢ`      | Change in network output when peer i is removed                  |
| `H(Qⱼ(x))`    | Fisher information (Hessian of loss) — curvature of loss surface |
| Sum over `Dⱼ` | Averaged across all data peer j sees                             |

**Intuition:** H measures how sensitive the loss is to changes. A peer that, if removed, causes large changes in a sensitive region of the loss is very valuable — high pruning score = high rank.

This borrows from **neural network compression** (LeCun 1989, Optimal Brain Damage) — same idea used to decide which weights to prune from a single network, applied here to entire peer models.

---

## 6. The Collusion Problem

### Why the ideal score breaks down

The blockchain can only see:

- The weight matrix `W` (what peers submit)
- Stake `S`

It **cannot** audit model parameters or verify peers actually computed equation (5) above.

### The attack

A group of colluding peers (a "cabal") can:

1. Set `wᵢⱼ = 1` for peers inside the cabal
2. Set `wᵢⱼ = 0` for all honest peers outside
3. Accumulate `R` (rank) and thus `I` (tokens) without contributing useful ML work

### Why this is the central problem

$$R = W^T \cdot S$$

If a cabal controls enough stake `S`, their internal votes dominate `R`, and they can farm emissions indefinitely. The rest of the paper (Yuma Consensus) is the proposed solution.

---

## 7. Key Relationships Summary

```
Dataset D → Loss L
Loss L + Stake S → Network Objective ∑Lᵢsᵢ
Peers evaluate each other → Weight matrix W
W + S → Rank R = WᵀS
Rank R → Incentive I
Incentive I → Stake update S(t+1) = S(t) + τI
```
