


# Kali — Scanning & Enumeration

> Enumeration = extract detailed info from services found in recon. Goal: find versions, users, shares, configs — anything useful for exploitation.

---

## Methodology

```
Open ports (from nmap)
    ↓
Identify services per port
    ↓
Enumerate each service
    ↓
Document versions, users, shares, paths
```

---

## Port → Service Cheatsheet

|Port|Service|Enumerate with|
|---|---|---|
|21|FTP|`ftp`, `nmap --script=ftp*`|
|22|SSH|banner grab, user enum|
|23|Telnet|`nc`, `telnet`|
|25|SMTP|`smtp-user-enum`, `nc`|
|53|DNS|`dig`, `dnsrecon`|
|80/443|HTTP/S|`gobuster`, `nikto`, `burp`|
|139/445|SMB|`enum4linux`, `smbclient`|
|111/2049|NFS|`showmount`|
|3306|MySQL|`mysql`, `nmap scripts`|
|3389|RDP|`xfreerdp`, `rdp-enum`|
|5985|WinRM|`evil-winrm`|
|27017|MongoDB|`mongo`|

## Service Enumeration

### FTP (21)

```bash
ftp 10.0.0.1                            # connect (try anonymous login)
nmap --script=ftp-anon 10.0.0.1         # check anonymous access
nmap --script=ftp-brute 10.0.0.1        # bruteforce
```

### SSH (22)

```bash
ssh user@10.0.0.1                       # connect
nc 10.0.0.1 22                          # banner grab — get version
nmap --script=ssh-auth-methods 10.0.0.1 # what auth methods allowed
nmap --script=ssh-brute 10.0.0.1        # bruteforce (slow)
```

### SMTP (25)

```bash
nc 10.0.0.1 25                          # banner grab
smtp-user-enum -M VRFY -U users.txt -t 10.0.0.1  # user enumeration
nmap --script=smtp-enum-users 10.0.0.1
```

### SMB (139/445)

```bash
enum4linux -a 10.0.0.1                  # full SMB enumeration
smbclient -L //10.0.0.1                 # list shares (no auth)
smbclient -L //10.0.0.1 -U user         # list shares (with user)
smbclient //10.0.0.1/share              # connect to share
nmap --script=smb-vuln* 10.0.0.1        # check for SMB vulns
nmap --script=smb-enum-shares 10.0.0.1  # enumerate shares
crackmapexec smb 10.0.0.1               # quick SMB info
crackmapexec smb 10.0.0.1 -u user -p pass --shares  # auth + shares
```

### NFS (2049)

```bash
showmount -e 10.0.0.1                   # list NFS exports
mount -t nfs 10.0.0.1:/share /mnt/nfs   # mount share
```

### HTTP/HTTPS (80/443)

```bash
nikto -h http://10.0.0.1                # web vuln scanner
whatweb http://10.0.0.1                 # identify tech stack
curl -I http://10.0.0.1                 # headers
```

### MySQL (3306)

```bash
mysql -h 10.0.0.1 -u root -p           # connect
nmap --script=mysql-info 10.0.0.1
nmap --script=mysql-databases 10.0.0.1 -p 3306
```

## Web Enumeration

### Directory & File Busting

```bash
gobuster dir -u http://10.0.0.1 -w /usr/share/wordlists/dirb/common.txt
gobuster dir -u http://10.0.0.1 -w /usr/share/wordlists/dirbuster/medium.txt -x php,html,txt
feroxbuster -u http://10.0.0.1 -w wordlist.txt     # recursive, faster
dirb http://10.0.0.1                                # simpler alternative
```

|Flag (gobuster)|Meaning|
|---|---|
|`dir`|Directory mode|
|`-u`|Target URL|
|`-w`|Wordlist|
|`-x`|File extensions to try|
|`-t`|Threads (default 10)|
|`-o`|Output file|
|`-k`|Skip TLS verification|

### Subdomain Busting

```bash
gobuster dns -d example.com -w /usr/share/wordlists/subdomains.txt
ffuf -w wordlist.txt -u http://example.com -H "Host: FUZZ.example.com"
```

### ffuf — Fast Web Fuzzer

```bash
# directory fuzzing
ffuf -w wordlist.txt -u http://10.0.0.1/FUZZ

# parameter fuzzing
ffuf -w wordlist.txt -u http://10.0.0.1/page?param=FUZZ

# POST body fuzzing
ffuf -w wordlist.txt -u http://10.0.0.1/login -X POST -d "user=FUZZ&pass=test"

# filter by response code / size
ffuf -w wordlist.txt -u http://10.0.0.1/FUZZ -fc 404
ffuf -w wordlist.txt -u http://10.0.0.1/FUZZ -fs 1234
```

## Vulnerability Scanning

### Nmap Vuln Scripts

```bash
nmap --script=vuln 10.0.0.1                 # general vuln scan
nmap --script=smb-vuln* 10.0.0.1            # SMB specific
nmap --script=http-shellshock 10.0.0.1      # shellshock
nmap --script=ftp-vsftpd-backdoor 10.0.0.1  # specific CVE
```

### Searchsploit

```bash
searchsploit apache 2.4                     # find exploits by name/version
searchsploit -m 12345                       # copy exploit to current dir
searchsploit --cve 2021-41773               # search by CVE
searchsploit -u                             # update database
```

### Nikto

```bash
nikto -h http://10.0.0.1                    # basic web scan
nikto -h http://10.0.0.1 -p 8080           # custom port
nikto -h http://10.0.0.1 -ssl              # HTTPS
nikto -h http://10.0.0.1 -o report.html    # save report
```

## Common Enumeration Workflow

```bash
# 1. identify open ports
nmap -sV -sC -p- 10.0.0.1 -oN scan.txt

# 2. web enumeration if 80/443 open
nikto -h http://10.0.0.1
gobuster dir -u http://10.0.0.1 -w /usr/share/wordlists/dirb/common.txt -x php,txt,html

# 3. SMB enumeration if 445 open
enum4linux -a 10.0.0.1
smbclient -L //10.0.0.1

# 4. check exploits for found versions
searchsploit apache 2.4.49
searchsploit vsftpd 2.3.4

# 5. NFS if 2049 open
showmount -e 10.0.0.1
```
